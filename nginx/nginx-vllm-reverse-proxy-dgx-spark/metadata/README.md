# Metadata

Debug information, configuration dumps, session logs, and any operational notes from troubleshooting or testing sessions.

## Purpose

This directory accumulates nginx reverse proxy metadata including:

- **Container log dumps** — generated by `05_b_dump_logs.sh` (saves to `nginx_logs_dump.txt`)
- **Configuration backups** — copies of `nginx.conf` or `docker-compose.yml` snapshots
- **Error logs** — captured from `logs/error.log` or Docker logs
- **Session notes** — troubleshooting documentation and testing results

## Usage

```bash
# Dump container logs to this directory
bash 05_b_dump_logs.sh

# Save configuration snapshot
cp nginx.conf metadata/nginx.conf.bak.YYYY-MM-DD
```

## Log Dump Location

When running `05_b_dump_logs.sh`, the log file `nginx_logs_dump.txt` is created in the **project root directory** (not here). For persistent metadata storage, you can move or copy dump files here manually.
